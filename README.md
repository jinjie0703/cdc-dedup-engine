# CDC Dedup Engine (内容可变分块去重存储引擎)

**CDC Dedup Engine** 是一个面向大文件的分布式去重存储引擎。不同于传统基于固定块（如 64KB）的切分方式，本引擎采用**可变分块（Content-Defined Chunking, CDC）**技术。当大文件局部修改发生时，分界点可随内容动态偏移，仅增量存储变动的数据块，从而在版本迭代和增量备份中极大节省带宽和存储空间。

## 🌟 核心特性

- **精准分块切分 (CDC)**：基于滚动哈希（Rolling Hash）算法动态确定分块边界，有效解决文件插删导致的哈希雪崩问题。
- **内容寻址存储 (CAS)**：数据块通过 `SHA-256` 指纹进行寻址，天然实现全局去重。
- **元数据本地持久化**：采用 SQLite 数据库管理文件根哈希（Tree Hash）、块索引树及数据块引用计数。
- **云原生对象存储扩展**：抽象 `StorageBackend` 接口，默认使用本地高性能存储，同时预留支持远端对象存储（如腾讯云 COS / 阿里云 OSS / S3）。
- **空间垃圾回收 (GC)**：基于引用计数的孤立数据块清理机制。

## 🛠️ 快速开始

### 1. 环境准备
本项目采用纯 Python 3 原生标准库开发（依赖 `sqlite3`, `hashlib`, `argparse`），无需额外安装第三方复杂的编译包，开箱即用！

### 2. 命令行操作 (CLI)

```bash
# 1. 存储文件并返回根哈希 (Root Hash)
python cli.py store ./sample_big_file.dat

# 2. 查看文件分块与去重统计详情
python cli.py stats ./sample_big_file.dat

# 3. 还原完整文件到指定路径
python cli.py fetch <root-hash> ./output_restored.dat

# 4. 执行垃圾回收 (清理无引用数据块)
python cli.py gc
```

### 3. 运行自动化基准测试
我们提供了一个一键自动化测试脚本，自动生成大文件、构造 100 字节局部修改，并对比系统实际存储与全量复制的空间占用差异：

```bash
python test_benchmark.py
```
