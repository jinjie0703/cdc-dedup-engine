# Code Architect & Documentation Sync Skill

针对 CDC Dedup Engine（Go 分布式去重后端 + React/Vite 看板前端）的高级架构设计与沉淀规范。

## 触发条件

1. **核心重构与扩展**：修改 CDC 切块算法参数、重构 CAS 存储接口、或对接远端对象存储（如阿里云 OSS / 腾讯云 COS）。
2. **前后端接口变更**：修改 Go 后端 API 协议或新增数据统计指标。

## 执行流程

### 第一阶段：架构提案（改代码前强制拦截）

在动手修改代码前，向用户输出简明的《技术方案说明》：

1. **设计思路**：遵循 Go 模块化接口设计（如 `storage.Backend` 抽象层）或 React 组件复用原则。
2. **影响范围**：列出将修改或新增的后端 `.go` 文件及前端 `.tsx` 文件。
3. **接口约定**：核心结构体、接口定义或 API 数据格式定义。
4. **等待确认**：输出完毕后询问用户：“是否同意该方案？”获得批准后方可执行修改。

### 第二阶段：文档沉淀与同步（开发完成后自动触发）

功能开发与测试验证完成通过后：

1. **同步 README.md**：将新特性要点写入项目概览或功能列表。
2. **同步设计文档**：若涉及架构变动，同步更新或记录关键数据流至 `DESIGN.md` 或相关技术文档。
