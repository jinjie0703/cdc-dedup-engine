# Self-Healing Build Skill (双栈自愈构建)

针对 Go 分布式去重后端与 React 前端项目的闭环错误修复机制。当构建或测试出现错误时，智能体必须主动接管修复，禁止直接把原始报错抛给用户。

## 触发条件

1. **后端报错**：执行 `go build`、`go test ./...` 或 `go run` 出现语法、类型或包依赖错误。
2. **前端报错**：执行 `npm run build`、`tsc -b` 或 Vite 开发服务器报出 TypeScript/CSS/资源打包错误。

## 执行步骤

1. **定位错误根源**：
   - 仔细分析终端命令输出的错误日志（文件路径、行号、错误码）。
   - 使用 `view_file` 准确定位到报错代码行及其上下文。
2. **执行严谨修复**：
   - **Go 后端**：注意指针接收者、接口实现完整性（如 `storage.Backend`）、错误处理（`if err != nil`）及包路径引用。
   - **React 前端**：严禁使用 `@ts-ignore` 或将变量强转为 `any` 敷衍了事，必须使用正确的接口定义（Interface/Type）或安全的类型推导。
3. **闭环验证循环 (The Loop)**：
   - 修改完成后，立刻在后台重新运行失败的构建命令（如 `go test ./...` 或 `npm run build`）。
   - 若仍有错误，循环执行定位与修复，直至零报错。
4. **汇报结果**：向用户总结修复的问题点，确认全栈构建畅通。
