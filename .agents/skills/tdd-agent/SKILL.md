# Test-Driven Dedup Engine Skill

针对数据去重引擎核心层（非 UI 看板）的测试驱动开发规范。去重算法对哈希精度、数据完整性和内存性能要求极高，任何核心逻辑修改必须有单元测试保障。

## 触发条件

在进行以下后端核心模块开发或修改时触发：

1. **切块算法层 (`pkg/chunker`)**：滑动窗口边界判断、分块大小约束（Min/Avg/Max Chunk Size）。
2. **内容寻址存储层 (`pkg/storage`)**：本地磁盘 CAS 读写、远端 OSS/COS 对象上传下载及垃圾回收 (GC)。
3. **元数据索引层 (`pkg/db`)**：SQLite 文件指纹映射、分块引用计数事务处理。

## 执行步骤

1. **防护网检查**：
   - 检查目标模块下是否已存在 `_test.go` 文件。如果不存在，立刻为其编写基础测试框架。
2. **红灯预警 (Write Failing Test)**：
   - 先编写针对新功能或修复边界的测试用例（例如：测试切块哈希碰撞、测试边界大小切块）。
   - 运行 `go test ./pkg/...` 确认用例能够准确捕捉到当前未实现或待修复的问题（呈现红灯 Fail）。
3. **绿灯通行 (Implementation & Pass)**：
   - 实现业务逻辑，再次运行 `go test ./...`，直至测试全部通过。
4. **防退化保障**：
   - 向用户汇报时，提供通过测试的命令日志，保障核心引擎去重逻辑“零退化”。
